SYSTEM INFORMACYJNO-UCZACY CITRONEX - JAK AKTUALIZOWAC

CEL
Pracownik ma dostac jeden link i otworzyc szkolenie w przegladarce telefonu.
System ma prowadzic czlowieka krok po kroku: mapa, wejscie, zasady, reader, rzed, wozek, przerwa, restart i test.

LINK DLA PRACOWNIKOW
https://oleksandrkiris.github.io/citronex-siechnice-szkolenie/

CO JEST WAZNE
1. Pracownik dostaje jeden link: index.html.
2. Technicznie system jest podzielony na lekkie moduly HTML, zeby telefon nie ladowal wszystkiego naraz.
3. Nie dodawac pol: imie, nazwisko, wybor etapu / miejsce jako formularz.
4. Nie dodawac gotowych prosb o pomoc do kopiowania.
5. Magazyn / sortownia nie ma readerow.
6. Nie zmieniac logiki instrukcji readera bez sprawdzenia z materialami zrodlowymi.
7. Jezyki maja zostac tylko na gorze strony.
8. Strona musi dobrze dzialac na malych telefonach.

JAK ZROBIC ARCHIWUM PRZED ZMIANA
Przed duza zmiana skopiuj aktualna wersje do folderu wersje.
Przyklad:
wersje/2026-07-01.html

JAK WGRAC ZMIANY
1. Zmien potrzebny modul HTML albo produkcyjne pliki:
   - assets/js/training-data.js
   - assets/js/training-app.js
   - assets/css/training.css
2. Sprawdz lokalnie na telefonowym rozmiarze ekranu.
3. Uruchom audyt:
   node tools/audit-project.js
4. Zrob commit na galezi main.
5. Wgraj main na GitHub.
6. Wgraj ta sama wersje na gh-pages.
7. Poczekaj 1-3 minuty.
8. Sprawdz publiczny link.

JAK SPRAWDZIC PO AKTUALIZACJI
- Strona sie otwiera.
- PL / EN / UA / RU / AZ / ES / FIL / ID / NE dzialaja.
- Mapy sa widoczne na poczatku.
- Zdjecia sie laduja.
- Reader jest zgodny z instrukcjami.
- Magazyn nie ma readera.
- Nie ma poziomego przewijania na telefonie.
- Nie ma pol imie / nazwisko.
- Nie ma gotowej prosby o pomoc.

JAK DODAC DO EKRANU TELEFONU
Android:
Otworz link w Chrome -> trzy kropki -> Dodaj do ekranu glownego.

iPhone:
Otworz link w Safari -> Udostepnij -> Do ekranu poczatkowego.

UWAGA
To jest strona w przegladarce. Przy pierwszym otwarciu potrzebny jest internet.
